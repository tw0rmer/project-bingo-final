Technical Implementation Details

Backend API Structure

GET /api/lobbies - List all lobbies
GET /api/lobbies/:id - Get lobby details
GET /api/lobbies/:id/participants - Get lobby participants
POST /api/lobbies/:id/join - Join lobby with seat selection
POST /api/lobbies/:id/leave - Leave lobby with refund

Frontend Stack

  • React 18: Functional components with hooks
  • TypeScript: Full type safety
  • Wouter: Lightweight routing
  • Tailwind CSS: Utility-first styling
  • React Query: Server state management

Backend Stack

  • Node.js + Express: API server
  • TypeScript: Type-safe backend
  • JWT: Authentication tokens
  • bcryptjs: Password hashing
  • Socket.io: Real-time communication

System Architecture & Data Flow

Authentication Flow

1
User Login: Credentials sent to /api/auth/login
2
Token Generation: JWT token created with user ID and email
3
Frontend Storage: Token stored in localStorage for persistence
4
API Requests: Authorization header with Bearer token

Lobby Join Process

Validation Steps
  1. 1. User authentication check
  2. 2. Lobby existence validation
  3. 3. Seat availability verification
  4. 4. Balance sufficiency check
  5. 5. Duplicate participation prevention
Transaction Flow
  1. 1. Entry fee deduction
  2. 2. Transaction record creation
  3. 3. Lobby seat update
  4. 4. Participant record creation
  5. 5. Frontend state refresh
